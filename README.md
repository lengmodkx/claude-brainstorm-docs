# 头脑风暴设计文档

此目录用于集中存储所有使用 `/brainstorm` 技能创建的设计文档。

## 目录结构

```
brainstorm-docs/
├── README.md              # 本文件
└── YYYY-MM-DD-<topic>-design.md   # 设计文档
```

## 文件命名规范

设计文档按照以下格式命名：`YYYY-MM-DD-<topic>-design.md`

- `YYYY-MM-DD`: 创建日期
- `<topic>`: 设计主题或功能名称
- 示例：`2026-01-21-user-authentication-design.md`

## 设计文档内容

每个设计文档应包含：

1. **概述**: 功能/需求描述
2. **架构设计**: 系统架构和技术选型
3. **组件设计**: 各组件职责和交互
4. **数据流**: 数据在系统中的流动路径
5. **错误处理**: 异常情况和错误恢复策略
6. **测试策略**: 测试方法和覆盖范围
7. **实施计划**: 具体实施步骤

## 使用方法

当需要创建新的设计文档时：

1. 使用 `/brainstorm` 命令启动头脑风暴
2. Claude 会引导你完成需求分析和设计过程
3. 最终设计文档将保存到此目录

## 版本控制

建议将此目录纳入 git 版本控制，以便跟踪设计变更历史。
